Problema

La detección temprana de incendios es esencial para prevenir desastres, especialmente en áreas de difícil acceso. Los sistemas tradicionales suelen presentar alta latencia y dependen de sensores físicos que solo se activan cuando el fuego ya ha avanzado, lo que limita la capacidad de respuesta.

FireGuard AI propone una solución innovadora basada en visión artificial e inteligencia artificial (IA) para identificar fuego en tiempo real a través de la cámara de una laptop. Al detectar fuego, el sistema activa automáticamente una bomba de agua, enciende un LED rojo y un buzzer para alertar sobre el peligro. En ausencia de fuego, se ilumina un LED verde, indicando condiciones seguras.

Este enfoque combina IA y automatización para ofrecer una respuesta inmediata y eficiente, reduciendo daños materiales y salvando vidas. FireGuard AI representa un avance significativo frente a los métodos tradicionales, proporcionando una solución rápida, inteligente y confiable

Hardware y Materiales

ESP32

El cerebro del sistema. Microcontrolador de alto rendimiento con conectividad Wi-Fi y Bluetooth integrada, encargado de recibir las señales de la IA y controlar los actuadores.

Actuadores

Sistema de respuesta física compuesto por un Driver L293D y una bomba de agua de 5V, activados automáticamente al detectar fuego confirmado.

Sistema de Alerta

Feedback audiovisual inmediato mediante un Buzzer piezoeléctrico y LEDs indicadores de estado (Rojo: Peligro, Verde: Seguro).

Detección de Fuego (Python - Visual Studio Code) 


import cv2
import serial
from ultralytics import YOLO

arduino = serial.Serial('COM5', 9600)

model = YOLO("best.pt")
cap = cv2.VideoCapture(0)

if not cap.isOpened():
    print("❌ No se pudo abrir la cámara")
    exit()

while True:
    ret, frame = cap.read()
    if not ret:
        print("❌ No se pudo leer frame")
        break

    # Predicción
    results = model.predict(frame, imgsz=640, conf=0.4)

    fire_detected = False

    for box in results[0].boxes:
        cls = int(box.cls)
        name = model.names[cls]
        if name.lower() == "fire":
            fire_detected = True
            break

    # Serial
    arduino.write(b"1\n" if fire_detected else b"0\n")

    annotated = results[0].plot()
    cv2.imshow("Detección de Fuego", annotated)

    if cv2.waitKey(1) & 0xFF == ord('q'):
        break

cap.release()
cv2.destroyAllWindows()
arduino.close()

Lecciones Aprendidas

Sincronización Serial

Lograr una comunicación estable y sin latencia entre la detección en Python y la actuación del ESP32 fue crítico. Implementamos un protocolo de handshake simple para asegurar que los comandos de activación lleguen correctamente.

Calibración Ambiental

El ajuste de umbrales de confianza en la cámara fue necesario para evitar falsos positivos en entornos con iluminación variable o fuentes de luz intensa que podían confundirse con fuego.

Funcionamiento y Prueba de Concepto en Vivo

Funcionamiento

Este sistema consta de tres componentes principales: un dispositivo ESP32, una laptop con cámara, y un conjunto de elementos de automatización física, incluyendo una bomba de agua, LEDs y un buzzer.

  1. Detección de fuego: En primer lugar, un modelo de IA previamente entrenado para identificar fuego es ejecutado en la laptop. La cámara de la laptop captura las imágenes del entorno, las cuales son procesadas por el modelo para detectar la presencia de fuego en tiempo real.
  2. Comunicación entre dispositivos: Si el modelo detecta la presencia de fuego, envía una señal al ESP32 a través de la conexión serial para que este active los mecanismos de respuesta.
  3. Activación de la bomba y señales visuales/auditivas: El ESP32, al recibir la señal de activación, enciende el LED rojo, lo que indica la presencia de fuego. Además, activa el buzzer para generar una alerta sonora. En paralelo, se activa la bomba de agua mediante un driver que controla su encendido, permitiendo el inicio del proceso de extinción del fuego.
  4. Condición de seguridad: Si no se detecta fuego, el sistema mantiene el LED verde encendido, indicando que todo está en orden. El buzzer permanece apagado, y la bomba de agua se mantiene inactiva para ahorrar energía y evitar falsos disparos.

Este sistema busca no solo ser una solución efectiva para la detección temprana de incendios, sino también demostrar la viabilidad de soluciones integradas de IA y automatización en el contexto de prevención de incendios.
